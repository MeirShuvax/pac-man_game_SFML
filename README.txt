 Creator: Meir Shuvax- Student in Hadassah Academic College Jerusalem. This project was made as final project in semester a' year b'. 

To build this game-project you must have SFML library downloaded on your computer and located in the C directory because that is where the cmake directs the compiler to find it. When you have it you can build it (cmake project building) and run/debug as you wish. 

SFML can be downloded from: https://www.sfml-dev.org/download.php
 
 
 בתרגיל זה נבנה משחק של סופר-פקמן, כולל משחק שלם שעובד, רץ, כולל גרפיקה, והכל 
כדי שירגיש משחק מוחשי ואמיתי, ושיהיה כיף לשחק בו.
 
יצרנו 33 מחלקות שונות, וכמובן קבצים של מימושיהם, כדלהלן:
Board.h - אחראי על הפונקציות שקשורות ללוח (-שלב המשחק הנוכחי), הדפסתו, שינוי וניהול דברים שקשורים ללוח המשחק.
Board.cpp -   אחראי על מימוש הפונקציות שקשורות ללוח (-שלב המשחק הנוכחי), הדפסתו, שינוי וניהול דברים שקשורים ללוח המשחק.
controller.h - אחראי על ניהול, תפעול, והרצת המשחק השוטף שיפעל כראוי.
controller.cpp - אחראי על מימוש הפונקציות שקשורות לניהול, תפעול, והרצת המשחק השוטף שיפעל כראוי.

Resources class - אחראי על ניהול המשאבים השונים שקשורים למשחק (כולל את font, sound, ו LoadingImages- שטוענים 
בהתאמה את כל משאבי המשחק השונים (לפי שמותיהם).

buttons class  - אחראי על מימוש הפונקציות שקשורות לניהול כפתורי התפריט, והגדרתם במידת הצורך.

- אחראים על מימוש וינהול כפתורי התפריט במידה והשחקן לחץ על אחד מהם (כל אחד לגופו).

שימוש במונחה עצמים, ירושות ופולימורפיזם, כדי לעשות את הקוד יותר פשוט וניתן לשינוי (אם נרצה להוסיף אובייקטים וכו לדוגמה) בקלות.
מרדף השדונים - הם יקבלו את מיקום הפקמן בכל רגע נתון - וירדפו אחריו לפי המיקום שלו בהתאמה (יתקדמו לפי ה row ו ה col - עקב זמן הריצה הגבוה 
של הרקורסיה - שהיא יותר מדוייקת לכאורה, עשינו את השיטת רדיפה הזאת, בעיננו היא ממש משחקית.)
השדונים - יש לנו 3 סוגי שדונים - השדון האדום - רץ במהירות גבוהה, הצהוב, במהירות בינונית, הירוק, במהירות נמוכה. 

 design :
Board  - תפקידו לנהל את לוח המשחק, ולבצע את הפעולות הנדרשות בו.
אובייקטי המשחק - כל האובייקטים במשחק מתחלקים לפי קטגוריות -נעים ושאינם- שהם יורשים מהאבות שלהם, שהאבות - יורשים ממחלקת object, והביצוע יתחלק בפועל לפי
הדברים המשותפים לכולם ושאינם.
כנ"ל - הסופר פקמן והפקמן - במידה ויש התנהגות שונה - ע"י פולירמופיזם, כל אחד יפנה להתנהגות המתאימה לו במידה זו.
כנ"ל - המחלקות השונות של סוגי המתנות - יירשו ממחלקת gift - וכל אחת תבצע את ההתנהגות המתאימה לה.
מחלקות המשאבים - כל אחת תעשה את תפקידה המתאים לה (סאונד, פונטים, טעינת תמונות)
מחלקת התפריט - כשמה כן היא - תעשה את כל הדברים שקשורים לתפריט בכלליות - 
ע"י שימוש במחלקות הכפתורים השונים לסוגיהם.
מחלקת ה controller- תנהל את המשחק, ותקרא לכל המחלקות השונות שיטפלו כל אחד לפי המקרה שלו.
מחלקת הזמן - תהיה אחראית על זמן המשחק.
